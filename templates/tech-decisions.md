# Technical Decisions

## How to Use
1. Before making architectural decisions, check existing decisions
2. Avoid contradicting previous decisions without discussion
3. If revisiting a decision, reference the original entry

## Entry Format
```
### [Date] - [Decision Title]
- **Context**: [why this decision was needed]
- **Options considered**: [alternatives]
- **Decision**: [what was chosen]
- **Reasoning**: [why]
- **Consequences**: [trade-offs, future implications]
- **Related files**: [affected files]
- **Revisited**: [date and reason, if applicable]
```
